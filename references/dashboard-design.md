# Dashboard 设计规范

## 整体风格

指挥中心式，深色主题，数据密度高但不压迫。参考 Linear、Vercel Dashboard 的设计语言。

字体：系统默认 sans-serif，中文使用系统字体栈。

## 布局结构

```
┌─────────────────────────────────────────┐
│  顶部导航：标题 + 搜索框 + 部署按钮        │
├─────────────┬───────────┬───────────────┤
│  研究摘要    │  主题趋势  │  月度产出统计  │
│  (本周)      │  (折线图) │  (柱状图)     │
├─────────────┴───────────┴───────────────┤
│  活跃热力图（GitHub 风格，按周显示）        │
├─────────────────────────────────────────┤
│  过滤器：全部 / 按项目 / 按主题 / 按类型   │
├─────────────────────────────────────────┤
│  文件卡片网格（3列）                       │
│  ┌──────┐ ┌──────┐ ┌──────┐             │
│  │ 卡片 │ │ 卡片 │ │ 卡片 │             │
│  └──────┘ └──────┘ └──────┘             │
└─────────────────────────────────────────┘
```

## 文件卡片

每张卡片显示：
- 文件类型图标（HTML / MD）
- 文件名
- 项目名 + 主题标签
- 描述（1行，超出省略）
- 收录时间
- 点击直接打开文件（使用绝对路径）

## 数据可视化

**主题趋势**：折线图，X轴为周，Y轴为各主题文件数量，最多显示前5个主题。

**活跃热力图**：类似 GitHub contributions，每格代表一天，颜色深浅代表当天收录数量，显示最近52周。

**月度产出**：柱状图，显示最近6个月每月收录数量。

## 搜索与过滤

- 顶部搜索框：实时过滤，匹配 filename、project、topic、description、tags
- 项目过滤：下拉多选
- 主题过滤：标签云点选
- 类型过滤：HTML / MD 切换

## 技术要求

- 纯静态 HTML，内联 CSS 和 JS，不依赖外部 CDN
- 数据直接内嵌在 HTML 的 JS 变量中（从 registry.json 读取后写入）
- 图表使用轻量级纯 JS 实现，不引入第三方库
- 支持本地打开（file:// 协议）和 GitHub Pages 部署
